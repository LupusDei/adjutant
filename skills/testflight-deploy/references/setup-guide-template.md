## Step 1: Register Your Bundle ID

4. **Access**: App Manager (minimum required for TestFlight uploads)

7. Note these values from the page:
   - **Key ID**: (shown next to the key name)
   - **Issuer ID**: (shown at the top of the keys list)

## Step 4: Set Up the Match Certificate Repository

Match stores encrypted code signing certificates and provisioning profiles in a private Git repo.

1. **Create a private repository** for certificates:
   - GitHub: https://github.com/new (set to **Private**)
   - Suggested name: `ios-certificates` or `match-certificates`
   - Repository URL: `{{MATCH_GIT_URL}}`

2. **Create an SSH deploy key** for CI access:
   ```bash
   ssh-keygen -t ed25519 -C "match-ci-deploy-key" -f match_deploy_key -N ""
   ```

3. **Add the public key** to the certificate repo:
   - Go to the cert repo → Settings → Deploy Keys → Add deploy key
   - Title: `GitHub Actions`
   - Key: paste contents of `match_deploy_key.pub`
   - Allow write access: NOT needed — read-only is sufficient

4. **Save the private key** (`match_deploy_key`) — you'll need it for GitHub secrets

## Step 5: Initialize Match (Run Once Locally)

This generates your code signing certificates and provisioning profiles, then encrypts and pushes them to your certificate repository.

```bash
# From your project root
bundle install
bundle exec fastlane match appstore
```

When prompted:
- **Passphrase**: Choose a strong passphrase and **save it** — needed for GitHub secrets
- **Apple ID**: Enter your developer account email if prompted
- **Password**: Your Apple ID password or app-specific password

This only needs to run once. Run it again if certificates expire (after 1 year).

---

## Step 6: Configure Xcode for Manual Code Signing

GitHub Actions cannot use automatic code signing. Switch to manual:

1. Open your project in Xcode
2. Select your **app target** in the project navigator
3. Go to **Signing & Capabilities** tab
4. **Uncheck** "Automatically manage signing"
5. For the **Release** build configuration:
   - **Provisioning Profile**: Select `match AppStore {{BUNDLE_ID}}`
   - **Signing Certificate**: Select `Apple Distribution`
6. For the **Debug** build configuration:
   - You can keep automatic signing for local development, or use `match Development`

> If you don't see the match profiles, run `bundle exec fastlane match appstore` locally first.

## Step 8: Trigger Your First Build

   git push origin main

5. After upload, TestFlight processing takes another **15-30 minutes**
6. Check **App Store Connect** → **TestFlight** to see your build

You can also trigger manually: **Actions** → **TestFlight Deploy** → **Run workflow**

## Troubleshooting

### "No signing certificate found" or "No provisioning profiles found"
- Run `bundle exec fastlane match appstore` locally to regenerate
- Verify `MATCH_PASSWORD` GitHub secret matches the passphrase you used
- Verify `MATCH_GIT_PRIVATE_KEY` has read access to the certificate repo

### "Could not create App Store Connect API token"
- Verify `ASC_KEY_ID` and `ASC_ISSUER_ID` are correct (check App Store Connect → Integrations)
- Verify `ASC_KEY_CONTENT` contains the full `.p8` file including header/footer lines
- Ensure the API key has at least "App Manager" access

### "The build number has already been used"
- The `latest_testflight_build_number + 1` strategy prevents this automatically
- If it occurs, manually increment the build number in Xcode, commit, and push

### "Provisioning profile doesn't match bundle identifier"
- Ensure the Bundle ID in Xcode exactly matches `{{BUNDLE_ID}}`
- Re-run `bundle exec fastlane match appstore` to regenerate profiles

### Expired provisioning profile (after ~1 year)
```bash
bundle exec fastlane match nuke appstore   # Revoke old certs/profiles
bundle exec fastlane match appstore        # Generate new ones
```
New certificates auto-propagate to CI via the match repo.

### Build succeeds but doesn't appear in TestFlight
- Processing can take 15-30 minutes — wait before assuming failure
- Check App Store Connect → TestFlight → iOS for processing status
- Verify the app record exists in App Store Connect (Step 2)

### macOS runner not available
- Check GitHub's runner documentation for current availability
- Update the `runs-on` value in `.github/workflows/testflight.yml`
- macos-14 = M1 (Xcode 15.x), macos-15 = M2 (Xcode 16.x)
